# FE-006 — Facebook OAuth button is a non-functional stub; no `AuthService.facebookLogin` exists

**Severity:** Low (feature gap; runtime safe — button does nothing)
**Component:** `src/components/LoginModal/LoginModal.tsx:187-190`, `src/services/api/auth.api.ts`
**Status:** Open

## Summary

The LoginModal renders a "Continue with Facebook" button, but the button has no `onClick` handler and there is no corresponding `AuthService.facebookLogin` method. Clicking it does nothing. The backend exposes `POST /auth/facebook` (accepts `{ facebook_token: string }`), so the server-side half is ready; only the client side is missing.

By contrast, Google OAuth is partially implemented: `<GoogleLogin>` from `@react-oauth/google` fires `AuthService.googleLogin(credential)` on success. Facebook has no equivalent.

## Backend contract

`POST /auth/facebook` expects:

```json
{ "facebook_token": "<FB_access_token>" }
```

Returns the same `ILoginResponse` shape as Google (`{ user, token }`).

## Expected

Clicking "Continue with Facebook" should:

1. Trigger the Facebook JS SDK (or `@greatsumini/react-facebook-login`) to obtain an access token.
2. Call `AuthService.facebookLogin(accessToken)` → `POST /auth/facebook` with body `{ facebook_token: accessToken }`.
3. On success: `authStore.login(response.user, response.token)`, persist to `localStorage`, close modal.
4. On failure: surface an inline error message (mirrors Google error handling at `LoginModal.tsx:204-206`).

## Suggested minimal implementation

```ts
// src/services/api/auth.api.ts — add alongside googleLogin:
facebookLogin: async (token: string): Promise<ILoginResponse> => {
  const response = await fetch(apiUrl("/auth/facebook"), {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ facebook_token: token }),
  });
  if (!response.ok) throw new Error("Facebook login failed");
  return response.json();
},
```

```tsx
// LoginModal.tsx — replace the dead button with a wired one:
import FacebookLogin from "@greatsumini/react-facebook-login";

<FacebookLogin
  appId={process.env.NEXT_PUBLIC_FACEBOOK_APP_ID ?? ""}
  onSuccess={async ({ accessToken }) => {
    try {
      setIsLoading(true);
      const response = await AuthService.facebookLogin(accessToken);
      authStore.login(response.user, response.token);
      window.localStorage.setItem("auth_token", response.token);
      window.localStorage.setItem("auth_user_id", String(response.user.id));
      setIsSuccess(true);
      setTimeout(onClose, 1500);
    } catch {
      setError("Facebook login failed. Please try again.");
    } finally {
      setIsLoading(false);
    }
  }}
  onFail={() => setError("Facebook login failed. Please try again.")}
  render={({ onClick }) => (
    <button
      className="bg-[#1877f2] text-white w-full p-3 rounded cursor-pointer mb-3 text-[15px] font-semibold border-none flex items-center justify-center gap-2"
      onClick={onClick}
    >
      <FaFacebook size={20} />
      Continue with Facebook
    </button>
  )}
/>
```

`NEXT_PUBLIC_FACEBOOK_APP_ID` needs to be added to `.env.local` and CI secrets.

## Note

Google OAuth shares a related Playwright test-isolation issue (tracked as part of FE-001, Root A): `<GoogleLogin>` renders inside a cross-origin iframe, making Playwright selectors unreliable. When Facebook is wired up, consider wrapping it in the same `data-testid` strategy recommended in FE-001 so both providers are testable without `frameLocator` acrobatics.
