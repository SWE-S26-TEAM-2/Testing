# FE-002 — Jest `Toggle.test.tsx` queries `getByRole("button")` but the component is now `role="switch"` (4 tests fail)

**Severity:** Medium (test rot; production behaviour correct, regression-detection lost)
**Component:** `src/__tests__/Toggle.test.tsx`, `src/components/Toggle/Toggle.tsx`
**Status:** Open

## Summary

`src/components/Toggle/Toggle.tsx:9-10` was changed (intentionally, for accessibility) to render with `role="switch"` and an `aria-checked` attribute. The corresponding jest tests in `src/__tests__/Toggle.test.tsx` still query the element with `getByRole("button")`, so the testing-library lookup throws "Unable to find an accessible element with the role 'button'" on every assertion.

## Failing tests (4)

- `Toggle Component › renders without crashing`
- `Toggle Component › renders with value=false — background should be #444`
- `Toggle Component › renders with value=true — background should be #ff5500`
- `Toggle Component › calls onChange when clicked`

## Reproduction

```bash
cd Frontend/media
npm test -- --testPathPattern=Toggle.test.tsx
```

## Expected

All 4 tests pass against the `role="switch"` Toggle.

## Actual

```
TestingLibraryElementError: Unable to find an accessible element with the role "button"
```

## Root cause

```tsx
// src/components/Toggle/Toggle.tsx:7-12
<div
  className={...}
  onClick={onClick}
  role="switch"
  aria-checked={value}
  data-state={value ? 'on' : 'off'}
>
```

vs

```tsx
// src/__tests__/Toggle.test.tsx:7,12,18,25
expect(screen.getByRole("button")).toBeInTheDocument();
const button = screen.getByRole("button");
```

## Suggested fix

Update the test to match the new role + assert `aria-checked` instead of inspecting inline `style.background`:

```tsx
import { render, fireEvent, screen } from "@testing-library/react";
import Toggle from "../components/Toggle/Toggle";

describe("Toggle Component", () => {
  it("renders without crashing", () => {
    render(<Toggle value={false} onClick={() => {}} />);
    expect(screen.getByRole("switch")).toBeInTheDocument();
  });

  it("reflects value=false with aria-checked='false'", () => {
    render(<Toggle value={false} onClick={() => {}} />);
    expect(screen.getByRole("switch")).toHaveAttribute("aria-checked", "false");
  });

  it("reflects value=true with aria-checked='true'", () => {
    render(<Toggle value={true} onClick={() => {}} />);
    expect(screen.getByRole("switch")).toHaveAttribute("aria-checked", "true");
  });

  it("calls onClick when clicked", () => {
    const handleClick = jest.fn();
    render(<Toggle value={false} onClick={handleClick} />);
    fireEvent.click(screen.getByRole("switch"));
    expect(handleClick).toHaveBeenCalledTimes(1);
  });
});
```

## Note for the FE team

This bug exists because **two streams of work touched Toggle without sync** — the QA branch added `role="switch"` for Playwright queryability, and a jest commit on main was authored against the old `role="button"`. Either the `role="switch"` change should be merged back fully (preferred — a11y is correct) or the QA branch should revert it. Recommend keeping `role="switch"` and updating the tests as above.
