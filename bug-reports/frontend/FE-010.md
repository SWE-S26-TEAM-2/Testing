# FE-010 — Deploy E2E feed-interaction tests fail when authenticated account has empty feed

**Severity:** Medium (blocks 4 deploy E2E assertions tied to track presence)
**Component:** Frontend feed page `/stream` + test data setup for deploy account
**Status:** Open

## Summary

In deploy-mode Playwright runs (`npm run test:e2e:real`), four tests that require at least one visible track card fail when the authenticated user has an empty feed.

The page loads successfully, but there are no track cards to interact with, so like/overflow/playback assertions fail.

## Failing tests (4)

- `Social and Library Interactions › user can like a track from the feed`
- `Social and Library Interactions › feed track overflow menu exposes add-to-queue`
- `Global Audio Player › clicking play on a feed track activates pause in the footer player`
- `Global Audio Player › footer transport toggles playback state`

## Reproduction

```bash
cd Frontend/media
npm run test:e2e:real
```

## Expected

For deploy E2E accounts used in interaction/player scenarios, `/stream` should contain at least one playable track card so social and playback actions can be exercised.

## Actual

`/stream` renders feed shell/empty state with zero track cards for the test account, causing interaction/player tests to fail due to missing actionable targets.

## Notes

- This is primarily a deploy-data readiness issue, not a framework crash.
- QA mitigation is in place to skip track-dependent checks when feed is empty.
- Long-term fix options:
  1. Maintain a seeded deploy QA account that follows artists with public tracks.
  2. Add a dedicated backend fixture/setup endpoint for test data seeding.
  3. Keep these specific checks in mock-data suites only.
