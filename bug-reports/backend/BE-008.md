# BE-008 — `POST /playlists/{id}/like` returns 400 on a playlist the creator just made

**Severity:** High (playlist like workflow broken for playlist owner; blocks E2E playlist test)
**Component:** `app/routers/playlist.py` (like endpoint), `app/services/playlist_service.py`
**Status:** Open — discovered via real-backend Playwright run (2026-04-29).

## Summary

Immediately after creating a playlist via `POST /playlists/`, calling `POST /playlists/{id}/like` on the same playlist returns:

```
HTTP 400
{"detail": "You already liked this playlist"}
```

The playlist was just created — the user never explicitly liked it. The backend appears to auto-insert a like record for the creator at creation time. When the user then tries to like their own new playlist, it fails because the record already exists.

## Reproduction

```bash
TOKEN=$(login as any user)

# Create playlist
PL_ID=$(curl -s -X POST .../api/playlists/ \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"name":"test","is_private":false}' | jq -r '.data.playlist_id')

# Like it → should succeed, but returns 400
curl -s -X POST .../api/playlists/$PL_ID/like \
  -H "Authorization: Bearer $TOKEN"
# → {"detail":"You already liked this playlist"}

# Unlike works fine
curl -s -X DELETE .../api/playlists/$PL_ID/like -H "Authorization: Bearer $TOKEN"
# → HTTP 200

# Like after unlike works
curl -s -X POST .../api/playlists/$PL_ID/like -H "Authorization: Bearer $TOKEN"
# → HTTP 200 {"data":{"playlist_like_id":...}}
```

## Two separate issues

1. **Auto-like on creation**: Playlist service silently inserts a like for the creator when the playlist is created. This is undocumented behavior and surprises the caller.
2. **Wrong status code**: "Already liked" should return **409 Conflict**, not **400 Bad Request**. 400 means the request was malformed; 409 means a state conflict exists. This affects clients that distinguish the two codes.

## Expected behavior

Option A (preferred): Remove the auto-like-on-creation behavior; let users explicitly like.  
Option B: Keep auto-like but document it, and change the duplicate-like response from 400 → **409**.

## Impact

- Any user trying to like their own new playlist gets a confusing "already liked" error.
- E2E playlist test fails: `expect([200,201,204,409]).toContain(400)`.
