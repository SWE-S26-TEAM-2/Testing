# BE-009 — CORS config only allows `localhost:3000`; blocks all other frontend origins

**Severity:** High (all browser-rendered page tests fail with "Failed to fetch"; production will break any non-3000 deployment)
**Component:** `app/main.py:68` — `CORSMiddleware(allow_origins=["http://localhost:3000"])`
**Status:** Open — discovered via real-backend Playwright run (2026-04-29).

## Summary

The backend's CORS middleware only permits one origin: `http://localhost:3000`.

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],
    ...
)
```

Any request from a different origin — including the test frontend on `http://127.0.0.1:3101`, the deployed frontend on a production domain, or the mobile app — will be rejected with a CORS error. The browser shows "Failed to fetch" with no useful error detail.

## Impact observed in E2E tests

Playwright tests that do `page.goto('/some-page')` and then the page makes API calls hit CORS. Every page-navigation-based test fails while API-only tests (Node.js `fetch`, no browser CORS enforcement) pass:

- `profile-real.spec.ts`: "Failed to fetch" → Follow button never appears → timeout
- `playlist-real.spec.ts`: page.goto profile page → "Failed to fetch"
- `upload-real.spec.ts`: `/creator/upload` page → blocked from uploading
- `track-real.spec.ts`: `/track/{id}` page → blocked from fetching track data
- `search-real.spec.ts`: `/discover` page → blocked from search API

## Verification

```bash
curl -I -X OPTIONS https://streamline-swp.duckdns.org/api/users/me \
  -H "Origin: http://127.0.0.1:3101" \
  -H "Access-Control-Request-Method: GET"
# → No Access-Control-Allow-Origin header returned
# → Browser blocks the request
```

## Fix

Replace the single hardcoded origin with the full list of allowed origins:

```python
ALLOWED_ORIGINS = [
    "http://localhost:3000",
    "http://localhost:3001",
    "http://127.0.0.1:3000",
    "http://127.0.0.1:3101",   # Playwright real-mode dev server
    "https://streamline-swp.duckdns.org",  # deployed frontend
    # add production domain when known
]

app.add_middleware(
    CORSMiddleware,
    allow_origins=ALLOWED_ORIGINS,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

Or use `allow_origin_regex` with a pattern covering `localhost` and the production domain.
