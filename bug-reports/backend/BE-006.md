# BE-006 — Audio `Range` edge cases (416 over-EOF, 404 file-missing-on-disk) untested at HTTP layer

**Severity:** Low (coverage gap; code paths exist and are correct)
**Component:** `app/routers/track.py:114`, `app/services/track_service.py:509-601`
**Status:** Open (coverage gap)

## Summary

`GET /tracks/{id}/audio` correctly implements two important error branches but neither has an HTTP-level test:

1. **416 Requested Range Not Satisfiable** — when client sends `Range: bytes=999999999-` past the end of the file. `track_service.py:601` raises `HTTPException(status_code=status.HTTP_416_REQUESTED_RANGE_NOT_SATISFIABLE)`.
2. **404 Not Found** — when the DB row exists but the audio file has been removed from disk (e.g. orphaned upload after a partial deletion).

Existing tests cover only the happy path: `Range: bytes=0-3` → 206 with first 4 bytes. See `tests/unit/test_track_router.py:452-459`.

## Why this matters

Streaming endpoints are heavy paths. The 416 branch protects against malformed clients (Safari's media engine has been known to retry with bad ranges); the 404 branch is the only safety net against silently serving a 200-with-empty-body when the underlying file was deleted out-of-band. Without tests, a refactor (e.g. switching to `FileResponse` or to S3) could regress these paths without anyone noticing.

## Verification of the gap

```bash
grep -nE "Range.*bytes=999|status_code=416|requested_range" tests/ -r
# → 0 results in tests
grep -nE "416|REQUESTED_RANGE" app/services/track_service.py
# → app/services/track_service.py:601 (production code raises 416)
```

def test_audio_range_past_eof_returns_416(client, monkeypatch):
    track_id = uuid4()
    fake_track = make_fake_track(audio_path="/tmp/test.mp3")
    monkeypatch.setattr(TrackRepository, "get_by_id", lambda db, tid: fake_track)
    write_bytes("/tmp/test.mp3", b"0123456789")  # 10-byte file
    r = client.get(
        f"/tracks/{track_id}/audio",
        headers={"Range": "bytes=999999999-"},
    )
    assert r.status_code == 416
    assert r.headers.get("Content-Range") == "bytes */10"

def test_audio_404_when_file_missing_on_disk(client, monkeypatch):
    track_id = uuid4()
    fake_track = make_fake_track(audio_path="/tmp/does-not-exist.mp3")
    monkeypatch.setattr(TrackRepository, "get_by_id", lambda db, tid: fake_track)
    r = client.get(f"/tracks/{track_id}/audio")
    assert r.status_code == 404

def test_audio_full_response_with_no_range_returns_200(client, monkeypatch):
