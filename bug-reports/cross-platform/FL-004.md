# FL-004 — `social_buttons_test.dart` only asserts placeholder snackbar text; Google/Facebook auth outcomes never verified

**Severity:** Low (test coverage gap; production behaviour correct, regression-detection weak)
**Component:** `test/widgets/social_buttons_test.dart`
**Status:** Open

## Summary

`SocialButtonsWidget` tests verify that tapping the Google and Facebook buttons shows a snackbar. The snackbar text is `'Google sign-in coming soon!'` / `'Facebook sign-in coming soon!'` — these are placeholder messages. When a real OAuth implementation lands, the tests will pass against the placeholder but miss regressions in actual auth integration.

```dart
// test/widgets/social_buttons_test.dart:55-70
testWidgets('Google button is tappable and shows placeholder message', (tester) async {
  ...
  await tester.tap(find.text('Continue with Google'));
  await tester.pumpAndSettle();
  expect(find.text('Google sign-in coming soon!'), findsOneWidget);
});
```

## Why this matters

Once `lib/widgets/social_buttons.dart` is wired to `AuthService.googleLogin` (equivalent to `LoginModal`'s `<GoogleLogin>` component on the web), the snackbar text will change. Tests will either break (good — forces update) or the widget won't be wired yet (current state). Without a real assertion they provide no signal that OAuth actually works in the Flutter app.

## Verification

```bash
grep -n "coming soon\|placeholder" test/widgets/social_buttons_test.dart
# → 'Google sign-in coming soon!', 'Facebook sign-in coming soon!'
```

```bash
grep -n "googleLogin\|facebookLogin\|AuthService" lib/widgets/social_buttons.dart
# → 0 results — widget shows SnackBar with placeholder text only
```

This is lower priority than FL-001/FL-002/FL-003. The placeholder snackbar is a conscious design choice while OAuth is being implemented. Flag for the team so it doesn't get forgotten when real auth lands.
