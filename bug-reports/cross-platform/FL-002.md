# FL-002 — `mini_player_integration_test.dart` uses `track.id` which no longer exists (renamed to `trackId`)

**Severity:** Medium (test-side compile error; production code correct)
**Component:** `integration_test/player/mini_player_integration_test.dart:23`
**Status:** Open (blocked by FL-001; fixable in same pass)

## Summary

The integration test for the mini player references `track.id` at line 23. The `Track` model no longer has an `id` field — it was renamed to `trackId` in `feature/cross-tracks`. This is a subset of FL-001 but worth tracking separately as a QA-authored test that needs updating once FL-001 is fixed.

## Verification

```bash
grep -n "\.id\b" integration_test/player/mini_player_integration_test.dart
# → line 23: track.id
```

```bash
grep -n "trackId\|\.id\b" lib/models/track.dart
# → line 30: final String trackId;   (no bare .id)
```

## Fix

```dart
// integration_test/player/mini_player_integration_test.dart:23
// Before:
final id = track.id;

// After:
final id = track.trackId;
```

## Note

The QA `test/widgets/mini_player_test.dart` was already updated in this branch to use `trackId` (via `_makeTrack()` helper). The integration test file still has the old reference.
